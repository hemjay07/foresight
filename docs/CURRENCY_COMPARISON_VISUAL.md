## ANSWERING OBJECTIONS

### "But Won't Players Abuse Price Volatility?"

**Objection:** What if SOL is $10 today and $40 tomorrow? Fair?

**Answer:**
- Users get the same SOL either way (tokens, not fiat value)
- This is no different from traditional poker (chips have fixed value regardless of external markets)
- Display both: "0.01 SOL (≈$300 at current price)"
- Users are OK with this (crypto natives live with volatility)

**Fake currency makes it worse** (you have to manage exchange rates dynamically)

---

### "But We Want to Control the Economy"

**Objection:** We need inflation/deflation mechanics like a real game.

**Answer:**
- That's a future problem at 50K+ users
- For MVP (1-5K users), contest rake is enough revenue
- Don't over-engineer a problem you don't have yet
- If you add seigniorage later, it feels like you're diluting players (bad)

**Custom currency on launch looks like** you're planning to dilute players from day 1

---

### "But DeFi Games Use Tokens"

**Objection:** Yield farming games use custom tokens; why can't we?

**Answer:**
- Yield farming is a different game (money-printing machines)
- Those games explicitly ask users to speculate on token price
- Fantasy sports is skill-based, not speculative
- Players want to play, not manage a token portfolio

**Foresight is NOT a DeFi game. It's a gaming platform. Different rules apply.**

---

### "What About Regional Pricing?"

**Objection:** SOL costs different in different countries. Should we have local currencies?

**Answer:**
- USDC stablecoin exists (1:1 USD)
- Players can buy USDC on local exchanges
- SOL is fungible globally (same price everywhere on exchanges)
- Traditional fantasy sports use USD globally (no localization needed)

**Not a real problem. Don't invent complexity to solve it.**
