## 1. CONTEST DESIGN & ECONOMICS

### 1.1 — Is 3 Days Right?

**Recommendation: YES, 3 days is optimal for launch.** Here's why:

| Duration | Pros | Cons |
|----------|------|------|
| **1 day** | Maximum urgency | No time for word-of-mouth, too small sample |
| **3 days** | ✓ 72h window for viral spread | ✓ Meaningful scoring sample | ✓ Fits weekly reset model | ✓ Feels "live event" | Requires ~200 signups min |
| **7 days** | Longer runway | Less urgency, fatigue sets in |
| **14 days** | Boring, "always on" | Players lose interest mid-contest |

**Data:** Crypto gaming launches (Hamster Kombat, Notcoin, Ton Station) show 48-72h contests create peak social velocity. 3-day arc also matches weekly fantasy sports cadence, so it feels native to users.

### 1.2 — Prize Pool Structure: $100 Split

**Recommendation: 100/75/50 for top 3.**

```
Total: $225 (free entry, all prize pool splits)
├─ 1st Place: $100 (44%)
├─ 2nd Place: $75 (33%)
└─ 3rd Place: $50 (22%)
```

**Why this split:**
- **Industry standard** for tournaments (confirmed across poker, esports, prediction markets)
- **Incentivizes excellence** — 50% rewards winner, not participation trophies
- **Builds narrative** — "Can you crack top 3?" is simpler than spread-out payouts
- **Shows credibility** — Actual cash ($20+ min) proves you'll ship prizes and can afford mainnet

**Alternative considered:** 50/35/15
- Pro: Slightly more for 2nd place (feel-good)
- Con: Reduces 3rd place to $15 (less meaningful), less standard in gaming

**Devnet consideration:**
- Prize denominated in SOL on devnet (~$0.05-$0.10 equivalent, shows value)
- All prizes are **airdropped directly to winners' wallets** (proof of concept)
- No need to launch token yet (product first, momentum first)

## 4. REFERRAL PROGRAM STRATEGY

### 4.1 — Referral Mechanics (What's Already Built)

Your backend has a sophisticated referral system. Here's what we'll activate:

**Double-sided rewards:**
```
Referrer signs up → earns +100 XP
Referee signs up via code → earns +50 XP (bonus for them too!)
Referee drafts team → Referrer +50 XP
Referee completes week → Referrer +100 XP (only if referee stays active)
```

**Quality score:** Only active referrals count toward recruiting rank. Prevents farming.

**Milestones (5 tiers):**
| Milestone | Threshold | XP Bonus | Hint |
|-----------|-----------|----------|------|
| Recruiter | 1 referral | 50 XP | First friend |
| Talent Scout | 5 referrals | 100 XP | "You're spotting winners" |
| CT Influencer | 25 referrals | 500 XP | "You're a tastemaker" |
| Kingmaker | 50 referrals | 1000 XP | "You built this community" |
| Legend | 100 referrals | 2500 XP | "You changed the game" |

### 4.2 — Launch Referral Incentive (Days 0-3)

**First 3 days: 5x XP bonus for ALL referrals**

```
Normal signup XP: 100
Launch bonus: 100 × 5 = 500 XP per referral

Effect: Early players who share aggressively build 5000+ XP lead
This locks in "founding members" — founders see 15-25 referrals in week 1
```

**Why this works:**
- Scarcity (bonus expires at contest end)
- Compound incentive (early players get compounding XP)
- Community feedback: "I wish I got in earlier" (drives day 4+ signups)

### 4.3 — Viral Loop Design

**Goal:** Turn "I won $50" into "I referred 10 friends who won $20 each"

**Sequence:**
1. **Win → Share** (post-contest, when you see your prize claimed)
   - Built-in share buttons: X, Discord, copy link
   - Pre-filled: "I placed #1 in @ForesightCT's draft competition! 🏆 [prize amount] won."
   - CTA: "Invite friends to play"

2. **Share → Friend Clicks → Signup** (frictionless)
   - Link contains referral code (already built: `?ref=CODE`)
   - Referral code pre-fills in signup (need to verify this works)
   - Friend sees: "Playing with [your name]" during signup

3. **Signup → Draft → Play**
   - New user sees referrer's team on leaderboard or profile
   - "Who drafted you in? [Referrer name]" card
   - Encourages rivalry / camaraderie

4. **Play → Share Again** (rinse, repeat)
   - Each milestone achieved (draft complete, week 1 score, top 10, etc.) triggers "share your achievement"
   - Referred friends get 50 XP bonus visible on profile
   - "You earned 50 XP because [Referrer] invited you!"

### 4.4 — Anti-Farming Safeguards

**Risk:** Bots or farmers create 100 accounts, all refer each other.

**Mitigations (already in code):**
- ✓ Active 7-day requirement (only recent users count toward active_referral_count)
- ✓ Quality score (only engaged referrals count toward milestones)
- ✓ One-level depth (can't earn XP from "friends of friends")
- ✓ Event deduplication (can't claim signup bonus twice)

**New layer (recommend):**
- **Email verification required** (prevents mass account creation)
- **Wallet verification** (one wallet = one account, prevent multi-accounting)
- **Daily signup limit per referrer** (e.g., can refer max 10 per day, prevents bot blasts)

## 9. DEVNET-SPECIFIC RISK MITIGATION

### 9.1 — Likely Problems & Solutions

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| **Contest fails to lock** (cron error) | Medium | High | Pre-test admin endpoints. Have backup manual trigger. Notify players 1h before if manual lock needed. |
| **Scoring calculation breaks** (edge case) | Medium | High | Run scoring on test data pre-launch. Prepare rollback plan (re-score, correct scores manually if needed). Have audit trail. |
| **Prize claim fails** (SOL airdrop fails) | Low | Critical | Test airdrop pathway with real wallets (testnet SOL) 48h before. Have backup Phantom->wallet transfer if needed. |
| **Frontend crashes under load** (1000 concurrent) | Low | Medium | Vercel auto-scales. But test with load testing tool. Have degraded-mode (static leaderboard if DB slow). |
| **Influencers don't promote** | Medium | Medium | Outreach to 30 (expect 10-15 convert). If low, rely on organic growth + community. Still winnable. |
| **Contest gets exploited** (same player multiple teams) | Low | Medium | Only 1 entry per user per contest (already enforced). Check for duplicate wallets. Ban if found. |
| **Devnet SOL faucet runs out** | Very Low | Medium | Faucet is free, rarely empty. But if happens, communicate transparently. Use testnet SPL tokens as backup. |
| **X/Discord go down** (unlikely) | Very Low | Low | Have Telegram as backup channel. Publish on blog. Update in-game announcement. |

### 9.2 — Devnet Reputation Management

**Key message:** "This is a devnet preview, not mainnet. Expect some chaos. Report bugs, we'll fix them."

**When things break:**
1. **Acknowledge immediately** (within 1h)
2. **Explain clearly** ("Scoring cron failed due to DB timeout")
3. **Give ETA** ("Fixed in 2h, manual scoring will happen")
4. **Compensate if needed** (bonus XP for inconvenience, simple)
5. **Post-mortem** ("Here's what we learned, here's the fix")

**Public trust:** 100 players with 1 bad experience talk to 10. But 100 players with 1 problem that's **transparently fixed** talk to 100 positively.

## 14. THE SINGLE MOST IMPORTANT THING

**You have ONE asymmetric edge that FantasyTop didn't: the influencers themselves are the game.**

Every CT you're asking to play is a potential influencer in the game. When @delluminati drafts other CTs, they're incentivized to promote it because:
1. They want their followers to watch them play
2. They want high engagement scores (which come from their followers engaging on their posts)
3. Early-mover CTs will be "founders" when you ship the token

**This is meta-marketing at its best.** You're not paying influencers to promote a game. You're giving influencers a game they benefit from promoting.

**Lean into this hard.**
